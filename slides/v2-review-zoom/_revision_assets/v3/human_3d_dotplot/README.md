# Human 3D Dot Plot Candidate

Task: `review-zoom-v3-human-3d-dotplot-correlation`

## Answer

Yes, there is a human analog of the mouse zygotene dot plot, but the most conservative slide-ready version for the regular human Fig. 3/ED5 data is an **arm-pair** analog, not a raw per-PHR-pair claim.

Recommended candidate:

- `human_arm_pair_dotplot_candidate.png`
- `human_arm_pair_dotplot_candidate.pdf`

Use this as the main slide option. Each dot is an inter-chromosomal chromosome-arm pair. The x-axis is mean PHR Jaccard similarity for that arm pair, and the y-axis is 50 kb 3D contact frequency. The statistic printed on the plot is a **pointwise Spearman correlation across dots**, not a Mantel test.

## Unit Decision

The inspected human source data expose three related units:

| Unit | Available? | Comparable to mouse zygotene scatter? | Use here |
|---|---:|---|---|
| Arm-haplotype pair | Yes, in `*_contact_matrix.tsv` files such as HG002 Pore-C `77 x 77` matrices. | Partly. It is the contact-matrix unit used by Fig. 3, but sequence similarity is usually summarized after collapsing haplotypes to chromosome arms. | Used indirectly by the main contact matrices. |
| Arm pair | Yes, in `community_based/50000bp/*_phr_pair_correlation.tsv`. Despite the filename, the rows are `arm_a`, `arm_b`, `mean_jaccard`, `n_pairs`, `hic_contact`: one row per inter-chromosomal arm pair. | Strong main-data analog. It preserves the regular human Fig. 3/ED5 source context and gives a direct dot plot with correlation. | **Recommended main candidate.** |
| PHR sequence pair | Yes, but only in the community-free exclusion-control outputs under `exclusion_controls/no_strong/community_free/*_seqlevel.tsv`. | Closest schema to the mouse zygotene sequence-level scatter, but it is an exclusion-control analysis, not the main full human Fig. 3 input. | Provided as a labeled supplemental control only. |

So the human analog exists at arm-pair level for the regular main data. A PHR sequence-pair analog also exists, but it must be labeled as a `no_strong` community-free control.

## Sequence-Level Control

Supplemental output:

- `human_phr_sequence_pair_control.png`
- `human_phr_sequence_pair_control.pdf`

This uses the HPRCv2 community-free sequence-level output generated by `sequence_hic_correlation.py`. The unit is one pairwise comparison between two subtelomeric PHR sequences on different chromosomes. The contact value is the balanced contact per valid bin pair (`hic_contact_norm`), not a matrix-wide Mantel statistic.

| Dataset | Unit | n | Pointwise Spearman rho | p-value | Caveat |
|---|---:|---:|---:|---:|---|
| HG002 Pore-C 50 kb, sequence-level control | PHR sequence pair | 1,648 | 0.437 | 9.68e-78 | `no_strong` control excludes `chr4_q`, `chr10_q`, acrocentric p arms, and sex arms. |
| CHM13 Hi-C 50 kb, sequence-level control | PHR sequence pair | 370 | 0.785 | 1.65e-78 | Same `no_strong` control exclusion. |

This is closest to the mouse zygotene file schema, but it should not be described as the regular full human Fig. 3 dataset.
