Compute pairwise similarity between sequences in a region

Usage: impg similarity [OPTIONS]

Options:
  -h, --help
          Print help (see a summary with '-h')

Alignment input:
  -a, --alignment-files <ALIGNMENT_FILES>...
          Path to the alignment files (PAF, 1ALN, or TPA format)

      --alignment-list <ALIGNMENT_LIST>
          Path to a text file containing paths to alignment files (one per line, PAF, 1ALN, or TPA
          format)

Index options:
  -i, --index <INDEX>
          Path to the IMPG index file (implies single-index mode)

  -f, --force-reindex
          Force the regeneration of the index, even if it already exists

      --index-mode <INDEX_MODE>
          Index mode: single, per-file or auto (single, but per-file if >= 100 files)

          Possible values:
          - auto:     Auto-select based on file count
          - single:   Single combined index
          - per-file: One index per alignment file
          
          [default: auto]

      --unidirectional
          Disable bidirectional alignment interpretation (default: bidirectional enabled). By
          default, every alignment A->B creates implicit reverse mapping B->A, guaranteeing all
          genomes are bridges for transitive queries

Alignment options:
      --trace-spacing <TRACE_SPACING>
          Trace spacing for .1aln alignment files (used when converting tracepoints to CIGAR)
          
          [default: 100]

      --poa-scoring <POA_SCORING>
          POA alignment scores as match,mismatch,gap_open1,gap_extend1,gap_open2,gap_extend2
          
          [default: 5,4,6,2,24,1]

      --aligner <ALIGNER>
          Aligner to use for FASTA input
          
          [default: fastga]
          [possible values: fastga, wfmash]

      --fastga
          Use FastGA aligner (shorthand for --aligner fastga)

      --wfmash
          Use wfmash aligner (shorthand for --aligner wfmash)

      --fastga-frequency <FREQUENCY>
          FastGA k-mer frequency threshold (use k-mers occurring ≤ N times). Overrides
          --fastga-frequency-multiplier when set

      --fastga-frequency-multiplier <FASTGA_FREQUENCY_MULTIPLIER>
          FastGA k-mer frequency multiplier: when --fastga-frequency is unset, the effective
          frequency = `num_pansn_haplotypes * multiplier` (clamped to at least 10). Default 1
          reproduces FastGA's historical behavior of one k-mer hit per haplotype
          
          [default: 1]

      --map-pct-identity <MAP_PCT_IDENTITY>
          Minimum percent identity for wfmash mapping (e.g. "90" or ANI preset "ani50-2")

      --all-pairs
          Align all genome pairs separately (slower, uses more memory, but handles many genomes)

      --batch-bytes <BATCH_BYTES>
          Maximum sequence data per batch (e.g., "50M", "2G"). Partitions genomes into batches.
          Stored as a raw string so library consumers (like impg's `sweepga::align_self_paf`) can
          pass it verbatim; parse via `crate::cli::parse_metric_number` at use sites when you need a
          numeric byte count

      --batch-size <BATCH_SIZE>
          Explicit number of genomes per batch (manual override). Overrides --batch-bytes and
          --max-disk

      --max-disk <MAX_DISK>
          Maximum disk space for temporary files during alignment (e.g., "100G", "500M"). Computes
          batch size automatically to stay within budget. Overrides --batch-bytes

      --zstd
          Compress k-mer index with zstd for ~2x disk savings and faster I/O

      --zstd-level <ZSTD_LEVEL>
          Zstd compression level (1-19, higher = smaller files but slower). Default: 3
          
          [default: 3]

      --temp-dir <TEMPDIR>
          Temporary directory for intermediate files (defaults to TMPDIR env var, then current
          directory; use "ramdisk" for /dev/shm)

Query region:
  -r, --target-range <TARGET_RANGE>
          Target range in the format `seq_name:start-end`

  -b, --target-bed <TARGET_BED>
          Path to the BED file containing target regions

Filtering and merging:
  -d, --merge-distance <MERGE_DISTANCE>
          Required. Merge query-gathered ranges separated by at most this many bp. This also sets
          the largest internal gap/SV one query hop can absorb into one reported interval; larger
          gaps stay split. Use 0 for only overlapping/touching ranges, or --no-merge to disable
          merging

      --no-merge
          Explicitly disable merging for all output formats

      --min-result-identity <MIN_RESULT_IDENTITY>
          Minimum gap-compressed identity threshold (0.0-1.0)

  -l, --min-output-length <MIN_OUTPUT_LENGTH>
          Minimum output length: filter results shorter than this (bp)

      --subset-sequence-list <SUBSET_SEQUENCE_LIST>
          Path to a file listing sequence names to include (one per line)

      --consider-strandness
          Consider strandness when merging output intervals (defaults: merge strands for
          bed/gfa/maf; keep separate for fasta/fasta-aln)

Transitive queries:
  -x, --transitive
          Enable transitive queries (uses BFS by default, or DFS with --transitive-dfs)

Transitive query options:
      --transitive-dfs
          Use Depth-First Search instead of BFS for transitive queries (slower, but returns fewer
          overlapping results)

  -m, --max-depth <MAX_DEPTH>
          Maximum recursion depth for transitive overlaps (0 for no limit)
          
          [default: 2]

      --min-transitive-len <MIN_TRANSITIVE_LEN>
          Minimum region size to consider for transitive queries (default: 101; for fastga
          --approximate, set > trace_spacing)

      --min-distance-between-ranges <MIN_DISTANCE_BETWEEN_RANGES>
          Minimum distance between transitive ranges to consider on the same sequence
          
          [default: 10]

Coordinate options:
      --original-sequence-coordinates
          Update coordinates to original sequences when input sequences are subsequences
          (seq_name:start-end) for 'bed', 'bedpe', and 'paf'

Performance:
      --approximate
          Use approximate mode for faster queries with tracepoint files (.1aln/.tpa, only bed/bedpe
          output)

Output options:
      --distances
          Output distances instead of similarities

      --all
          Emit entries for all pairs of groups, including those with zero intersection

      --delim <DELIM>
          The part of each path name before this delimiter is a group identifier

      --delim-pos <DELIM_POS>
          Consider the N-th occurrence of the delimiter (1-indexed, default: 1)
          
          [default: 1]

      --reverse-complement
          Reverse complement reverse strand sequences (for 'fasta' output)

      --force-large-region
          Force processing of large regions (>10kbp) with maf/gfa output formats

      --gfa-engine <ENGINE[:WINDOW]>
          GFA engine: 'pggb' (default), 'seqwish', or 'poa'. Append ':WINDOW' to enable partitioned
          mode, e.g. 'pggb:10000' splits into 10kb windows, builds per-window, laces, and normalizes
          
          [default: pggb]

Sequence input:
      --sequence-files <SEQUENCE_FILES>...
          List of sequence file paths (FASTA or AGC) (required for 'gfa', 'maf', and 'fasta')

      --sequence-list <SEQUENCE_LIST>
          Path to a text file containing paths to sequence files (FASTA or AGC) (required for 'gfa',
          'maf', and 'fasta')

Basic filtering:
      --min-aln-length <BLOCK_LENGTH>
          Minimum alignment block length (omit to use aligner default)

      --num-mappings <NUM_MAPPINGS>
          n:m-best mappings kept in query:target dimensions. 1:1 (orthogonal), use ∞/many for
          unbounded. Default matches impg's historical pangenome setting: no mapping-axis filter
          before scaffolding
          
          [default: many:many]

      --overlap <OVERLAP>
          Maximum overlap ratio for plane sweep filtering
          
          [default: 0.95]

      --scoring <SCORING>
          Scoring function for plane sweep
          
          [default: log-length-ani]
          [possible values: ani, length, length-ani, log-length-ani, matches]

      --min-aln-identity <MIN_IDENTITY>
          Minimum per-mapping identity filter applied to PAF records after alignment (0-1 fraction,
          1-100%, or "aniN" for Nth percentile). This is a post-alignment PAF filter, not an
          alignment-stage knob — for wfmash's alignment-stage identity cutoff, use
          `--map-pct-identity`
          
          [default: 0]

      --self
          Keep self-mappings (excluded by default)

      --no-filter
          Disable all filtering

      --min-map-length <MIN_MAP_LENGTH>
          Drop mappings shorter than this before plane-sweep / scaffold filtering. Accepts k/m/g
          suffixes
          
          [default: 0]

Scaffolding and chaining:
      --scaffold-jump <SCAFFOLD_JUMP>
          Scaffold jump (gap) distance. 0 = disable scaffolding, >0 = enable (accepts k/m/g suffix).
          Default matches impg's historical 50kb — pass `--scaffold-jump 0` to turn scaffolding off
          
          [default: 50k]

      --scaffold-mass <SCAFFOLD_MASS>
          Minimum scaffold chain length (accepts k/m/g suffix)
          
          [default: 10k]

      --scaffold-filter <SCAFFOLD_FILTER>
          Scaffold filter mode: "1:1" (best), "M:N" (M per query, N per target), "many" (unbounded)
          
          [default: many:many]

      --scaffold-overlap <SCAFFOLD_OVERLAP>
          Scaffold chain overlap threshold for plane sweep filtering
          
          [default: 0.5]

      --scaffold-dist <SCAFFOLD_DIST>
          Maximum Euclidean distance from scaffold anchor for rescue (0 = no rescue, accepts k/m/g
          suffix)
          
          [default: 0]

      --min-scaffold-identity <MIN_SCAFFOLD_IDENTITY>
          Minimum scaffold identity threshold (0-1 fraction, 1-100%, "aniN", or defaults to
          --min-aln-identity)
          
          [default: 0]

      --scaffolds-only
          Output scaffold chains only (for debugging)

      --no-adaptive-scaffolds
          Disable scaffold-parameter adaptation to input sequence length.
          
          By default, when the FASTA average sequence length is known, `--scaffold-mass` is clamped
          to `avg_seq_len * 3/5` (nice-rounded) and `--scaffold-jump` is clamped to `avg_seq_len *
          10`. This is a no-op for whole-genome inputs (user values are already smaller than the
          clamps) but prevents default thresholds from filtering out everything when inputs are
          short (e.g. pangenome-window excerpts). Pass `--no-adaptive-scaffolds` to turn the
          clamping off.

Advanced filtering:
      --sparsify <SPARSIFY>
          Sparsification strategy: `none` (default = all pairs), `auto`, `<frac>` / `random:<frac>`,
          `giant:<prob>` / `connectivity:<prob>`, `tree:<near>[:<far>[:<random>]]` / `knn:…`,
          `wfmash:auto` / `wfmash:<frac>`. Parsed once by clap at CLI-parse time via
          `SparsificationStrategy::from_str`
          
          [default: none]

      --mash-kmer-size <MASH_KMER_SIZE>
          K-mer size used by the mash sketches driving tree/giant sparsification
          
          [default: 15]

      --mash-sketch-size <MASH_SKETCH_SIZE>
          Sketch size (number of minimizers) for the mash sketches driving tree/giant sparsification
          
          [default: 1000]

      --joblist
          Instead of running alignments, emit one shell command per genome pair to stdout (or
          `--output-file`) and exit. Useful for cluster / grid dispatch: each emitted line is a
          standalone `sweepga` invocation that aligns one pair. Requires multiple FASTA inputs

      --joblist-output-dir <JOBLIST_OUTPUT_DIR>
          Output directory for per-pair PAFs emitted by `--joblist`. Defaults to the current working
          directory. Ignored when `--joblist` is unset

      --ani-method <ANI_METHOD>
          Method for calculating ANI: all, orthogonal, nX[-sort] (e.g. n50, n90-identity,
          n100-score)
          
          [default: n100]

AGC archive options:
      --agc-prefix <AGC_PREFIX>
          Extract only samples matching this prefix (AGC input)

      --agc-samples <AGC_SAMPLES>
          Extract only these samples (comma-separated or @file with one per line)

      --agc-queries <AGC_QUERIES>
          Query samples for asymmetric alignment (prefix or comma-separated/@file list)

      --agc-targets <AGC_TARGETS>
          Target samples for asymmetric alignment (prefix or comma-separated/@file list)

      --agc-temp-dir <AGC_TEMPDIR>
          Temp directory for AGC extraction (defaults to --temp-dir)

Pair selection:
      --pairs <PAIRS_FILE>
          File of sample pairs to align (TSV: query<tab>target, one pair per line)

      --pairs-done <PAIRS_DONE>
          File to append completed pairs (for resume capability)

      --pairs-remaining <PAIRS_REMAINING>
          File to write remaining pairs after run (for resume capability)

      --list-pairs
          List all sample pairs and exit (for generating pairs files)

      --shuffle-pairs
          Shuffle pair order (for load balancing across runs)

      --shuffle-seed <SHUFFLE_SEED>
          Seed for pair shuffling (for reproducible ordering)

      --max-pairs <MAX_PAIRS>
          Maximum number of pairs to process in this run (0 = unlimited)
          
          [default: 0]

      --pair-start <PAIR_START>
          Start index for pair range (0-based, use with shuffle-seed for stable ordering)
          
          [default: 0]

      --sparsify-pairs <SPARSIFY_PAIRS>
          Sparsification strategy for pair selection (none, auto, random:<frac>, giant:<prob>,
          tree:<near>:<far>:<random>[:<kmer>])
          
          [default: none]

Graph induction (seqwish):
      --repeat-max <REPEAT_MAX>
          Maximum repeat count for transitive closure (0 = no limit)
          
          [default: 0]

      --min-repeat-dist <MIN_REPEAT_DIST>
          Minimum distance between repeats
          
          [default: 0]

      --min-match-len <MIN_MATCH_LEN>
          Minimum match length filter for alignments
          
          [default: 23]

      --sparse-factor <SPARSE_FACTOR>
          Sparse factor for input matches (0.0 = keep all)
          
          [default: 0]

      --transclose-batch <TRANSCLOSE_BATCH>
          Batch size for transitive closure computation
          
          [default: 10000000]

      --disk-backed
          Use disk-backed interval trees (slower but lower memory)

Graph smoothing (pggb engine):
      --target-poa-length <TARGET_POA_LENGTH>
          Comma-separated target POA block lengths in bp, one per smoothing pass (default:
          "700,1100" matching pggb's `-G 700,1100`)
          
          [default: 700,1100]

      --max-node-length <MAX_NODE_LENGTH>
          Maximum node length before chopping (default: 100)
          
          [default: 100]

      --poa-padding-fraction <POA_PADDING_FRACTION>
          Fraction of avg block length to use as POA padding (default: 0.001)
          
          [default: 0.001]

      --debug-dir <DEBUG_DIR>
          Save intermediate debug files (PAFs, FASTAs, sub-GFAs, chunk info) to this directory

PCA options:
      --pca
          Perform PCA/MDS dimensionality reduction on the distance matrix

      --pca-components <PCA_COMPONENTS>
          Number of PCA components to output (default: 2)
          
          [default: 2]

      --polarize-n-prev <POLARIZE_N_PREV>
          Number of previous regions to use for adaptive polarization (0 to disable)
          
          [default: 3]

      --polarize-guide-samples <POLARIZE_GUIDE_SAMPLES>
          Comma-separated names of the samples to use for adaptive polarization

      --pca-measure <PCA_MEASURE>
          Similarity measure to use for PCA distance matrix ("jaccard", "cosine", or "dice")
          
          [default: jaccard]

General options:
  -t, --threads <THREADS>
          Number of threads for parallel processing
          
          [default: 4]

  -v, --verbose <VERBOSE>
          Verbosity level (0 = error, 1 = info, 2 = debug)
          
          [default: 1]
